# ADR-0013: 초기 RSS 매체 셋 — 해외 6 + 국내 6

- Status: Accepted
- Date: 2026-05-22
- Deciders: yusik
- Related: [ADR-0005](./0005-data-sources-rss-google-trends.md), [ADR-0007](./0007-content-policy-headline-curation.md), [ADR-0008](./0008-category-taxonomy.md)

## Context (배경)

ADR-0005에서 데이터 소스를 **공식 RSS**로 한정했고, ADR-0008에서 8개 카테고리(top + 7 도메인)와 단일 분류·우선순위 룰을 박았다. 이제 v1에 실제로 등록할 매체를 정해야 한다.

요구사항:
- 영문 매체와 국내 매체의 **균형** (사용자 베이스가 둘 다 가능).
- **카테고리 커버리지** — 8개 카테고리 모두에 최소 1개의 매체가 매핑되어야 메인 페이지 카드 그리드가 비지 않음.
- **신뢰도** — 통신사·공영·전통 매체 우선. v1 출발에서 가십·연예 비중은 낮춤.
- **정치 다양성** (국내) — 진보·보수 한쪽으로 치우치지 않게 균형.
- **운영 부담 최소화** — 12개 내외에서 시작, 운영하며 PR로 조정.

## Decision (결정)

> 운영자 결정 (2026-05-22): **추천안 6+6 그대로 시작**. 운영 중 추가/제거는 PR로 처리.

### 카테고리별 매체 커버리지 (확인)

| 카테고리   | 해외 매체                         | 국내 매체              |
| ---------- | --------------------------------- | ---------------------- |
| `top`      | (자동 집계, ADR-0008)             | KBS                    |
| `world`    | BBC, Reuters, AP, Guardian        | 연합뉴스, KBS          |
| `politics` | Guardian                          | 연합뉴스, 한겨레, 조선  |
| `business` | BBC, Reuters, Bloomberg           | 연합뉴스, 조선, 한국경제 |
| `tech`     | BBC, The Verge                    | 연합뉴스, 전자신문      |
| `science`  | BBC, Guardian                     | (커버리지 ↓ — v2 보강) |
| `sports`   | (커버리지 ↓ — v2 보강)            | (커버리지 ↓ — v2 보강) |
| `culture`  | Guardian                          | 한겨레                  |

- **`science`, `sports`는 v1에서 커버리지가 얇음**. 운영 시 헤드라인이 너무 적게 잡히면 v2 ADR로 매체 추가.
- 카테고리별 RSS는 매체가 분리 제공할 때만 매핑. 통합 RSS만 주는 매체는 `top` 후보로 들어갈 수도 있음.

### 가중치 정책

- v1은 **모든 매체 weight = 1.0** (균등).
- 운영하며 dedupe 후 노출 빈도·품질 보고 PR로 weight 조정 가능.
- 가중치 변경은 ADR 신규 작성 없이 코드 변경으로 처리 (운영 튜닝 영역).
- 단, 가중치 체계 자체를 바꾸는 경우(예: 통신사·일반매체 차등 도입)는 별도 ADR로 supersede.

### 매체 추가/제거 절차

1. 후보 매체의 RSS URL을 직접 확인 — 200 OK + 유효한 XML + 최근 항목 1주 내 갱신.
2. `src/scraper/sources.ts`에 항목 추가 PR (혹은 제거 PR).
3. PR 설명에 매체명·강점 카테고리·정치 톤(국내) 명시.
4. 매체가 카테고리별 RSS 분리 제공 안 하면 `top` 후보로만 등록 또는 룰 기반 분류 (v2).

### RSS URL의 명시 위치

- RSS URL은 **본 ADR이 아니라 `src/scraper/sources.ts`에 박는다**. URL은 변동성이 있어 ADR로 박으면 자주 supersede가 필요 — 코드 영역에 두는 게 자연스러움 (ADR-0005 §매핑 원칙).
- ADR-0013은 "어떤 매체를 등록할지"의 결정만 박고, 구체 URL은 운영 동맥(코드)에 둠.

## Consequences (결과)

**긍정**
- 12개 매체로 시작 → 1인 운영 + GitHub Actions cron 5분 이내에 모두 처리 가능 (ADR-0006 흐름).
- 통신사(Reuters/AP/연합) 비중이 있어 신뢰도 베이스라인 확보.
- 정치 다양성(한겨레/조선) → 한쪽 편향 인상 회피.
- 커버리지 갭(science/sports)이 명시적으로 노출됨 → 운영 중 의도적 보강 가능.

**부정**
- `sports`와 `science`(국내) 매체가 없거나 약함 → v1 출발에서 해당 카테고리 카드가 적게 채워질 가능성. 메인 페이지 시각적 균형이 깨질 수 있음.
- Reuters RSS URL 변동성 — 매월 1회 정도 헬스체크 필요.
- 매체 12개는 RSS만 동작 가정. 1~2개가 일시 중단되면 카테고리 커버리지가 얇아짐 → 운영 모니터링 항목으로 흡수.

**중립**
- 한국 매체 RSS가 영문 매체보다 분리 단위가 거친 경우가 있음 (예: "사회"가 우리 카테고리에 없음 → ADR-0008의 우선순위 룰로 흡수).
- 통합 RSS만 주는 매체(예: 단일 "최신뉴스" RSS)는 v1에서 등록 안 함.

## Alternatives Considered (대안)

- **압축 4+4**: 가장 가볍지만 카테고리 커버리지 갭이 v1 출발부터 너무 큼.
- **광범위 10+10**: 다양성·매체 수 ↑. 그러나 RSS URL 헬스체크·dedupe 부담이 1인 운영에 비합리적. v2 이후 확장.
- **국내 우선 (8+4)**: 한국 사용자 우선이지만, 글로벌 트렌딩 키워드(ADR-0005) 표시 가치가 약해짐.
- **해외 우선 (4+8)**: 영문 RSS가 더 안정적이라 매체 운영 부담은 ↓. 그러나 사이트 정체성이 외신 큐레이션으로 좁아짐.
- **상용 API (NewsAPI 등)**: ADR-0005에서 이미 배제.
- **RSS URL을 ADR 본문에 명시**: URL이 변동성 높아 ADR이 매월 supersede 대상이 됨. 코드에 두는 게 운영 친화적.
